# 中国签证辅助申请系统 Demo 交互流程说明文档

> **文档版本**：v1.1  
> **信息来源**：2026年2月26日千代福技术部与开发团队需求对齐会议及《新版需求、知识库及比对逻辑》文档  
> **Demo 范围**：菲律宾马尼拉签证申请中心 · 商务签（M字签）& 过境签（G字签）

---

## 一、背景与产品定位

### 1.1 产品背景

原计划直接从领事馆获取申请数据以实现"智能审核"，但数据来源存在困难。因此调整思路：**让申请人自行通过 AI 辅助完成信息填写**，以此作为数据入口，并将整个产品形态定位为"辅助申请"服务。

核心目标仍是服务后续的"智能审核"能力，但面向申请人呈现的是一个 **便捷、低门槛的 AI 辅助签证申请体验**。

### 1.2 使用场景与表现效果

| 项目 | 说明 |
|------|------|
| 软件形态 | **网页界面**（布置于中心大厅外网电脑的成品软件） |
| 使用地点 | 菲律宾马尼拉中国签证申请中心（线下自助使用，工作人员简单指导） |
| 交互方式 | 鼠标点选 + 键盘文字输入，结合自然语言问答与单选形式 |
| 界面语言 | **英文**（产品设计阶段可先用中文，最终输出英文版） |

## 二、整体流程架构

```
【申请人进入系统】
        │
        ▼
┌───────────────────┐
│  Step 1           │  固定问询流程（不一定大模型驱动）
│  签证类型分流      │  → 确定赴华目的、入境需求，归属签证类型
└────────┬──────────┘
         │
         ▼
┌───────────────────┐
│  Step 2           │  传统OCR + 多模态大模型分析
│  材料一次性上传    │  → 自动提取上传文件关键信息，判断合规性
│  & 合规校验       │  → 提取必要的表格基础信息（约20%）
└────────┬──────────┘
         │ 校验不通过 → 终止并提示重新上传
         │ 校验通过 ↓
         ▼
┌───────────────────┐
│  Step 3           │  大模型驱动（分模块交互）
│  补充信息收集      │  → 以单选、问答方式补充缺失的必要信息
└────────┬──────────┘
         │ 随填随审，实时校验
         ▼
┌───────────────────┐
│  Step 4           │
│  信息汇总确认      │  → 向申请人展示匹配正式申请端的表单，请其确认
└────────┬──────────┘
         │
         ▼
┌───────────────────┐
│  Step 5           │
│  结构化数据输出    │  → 导出文件，由中心员工查阅授权后
│  （演示结果展示）  │  → 通过脚本平移至正式申请端
└───────────────────┘
```

## 三、各步骤详细说明

### Step 1：签证类型分流

**驱动方式**：固定问询流程（**不使用大模型**）  
**原因**：签证类型分类逻辑相对固定，使用大模型存在随机性偏差风险；采用固定选项可保证分类准确性。

**交互形式**：AI 提问 + 申请人点选（用户感知上是自然语言问答，底层是固定选项）

**问询步骤**：

| 轮次 | AI 提问示例 | 说明 |
|------|------------|------|
| Q1 | "What is the purpose of your trip to China?" | 判断签证大类：商务、过境、旅游等 |
| Q2 | "How many entries do you need?" / "How long do you plan to stay?" | 判断签证参数（入境次数、停留时间） |

**分流结果（Demo 范围）**：

- 赴华进行商贸活动、体育比赛等 → **M 字商务签**
- 途经中国前往第三国、海员过境等 → **G 字过境签**

---

### Step 2：材料上传与合规校验

**驱动方式**：传统 OCR + 多模态大模型分析  
**交互形式**：根据政策向申请人定向索要相关文件，申请人一次性上传。

#### 2.1 基础材料校验流程（所有类型通用）

任意一项校验不通过，则在该步骤终止流程并提示重新上传：

| 校验顺序 | 校验项 | 通过条件 / 政策要求 | 不通过处理 |
|---------|--------|------------------|-----------|
| 1 | **护照原件首页** | 有效期 ≥ 6 个月；至少 2 页空白页。如有双重/多重国籍，需提供所有有效国籍护照及旧护照。 | 提示护照不满足要求，终止流程 |
| 2 | **申请照片** | 6个月内拍摄，小2寸（48mm×33mm）、白底、正面、着非白色上衣。 | 提示重新上传合规照片 |
| 3 | **居留证明**（条件触发） | 若护照显示**非菲律宾公民**，需提交在菲居留、工作、学习的有效证明或签证。 | 提示补充居留证明材料 |
| 4 | **原中国签证复印件**（条件触发） | 系统询问是否曾获中国签证；如是，需提供最近一次获得的复印件。 | 未上传则当首次申请处理 |
| 5 | **原中国国籍核查**（条件触发） | 若出生地为中国大陆或港澳台，系统询问是否曾有中国国籍；如是，需提供原中国护照或有关证明。 | 按要求补充材料 |
| 6 | **未成年人材料**（条件触发） | 未满 18 岁需提供：出生证、父母护照复印件；如一方父母未同行，必须提供其签字的同意旅行函（Consent Letter）。 | 按要求补充材料 |
| 7 | **首次申请附加材料**（条件触发） | 首次赴华需提供：<br>1. **最近6个月银行存款证明及收费收据（必须）**<br>2. 任选其一：纳税证明+在职证明 / 商业登记证明 / 职业身份证 / 学生证等。 | 材料不完整则提示补充 |

#### 2.2 商务签（M 签）专项材料要求

| 材料项 | 详细合规要求 |
|--------|------------|
| 商务邀请函 | 须由中国境内商业贸易合作方出具，且包含以下完整内容：<br>1. **被邀请人信息**：姓名、性别、出生日期、护照号码等；<br>2. **访问信息**：来华事由、抵离日期、访问地点、与邀请人关系、费用来源等；<br>3. **邀请方信息**：邀请单位名称或邀请人姓名、联系电话、地址、单位印章、法定代表或邀请人签字等。 |

#### 2.3 过境签（G 签）专项材料要求（本次不涉及）

| 材料项 | 详细合规要求 |
|--------|------------|
| 联程机/车/船票 | 需提交前往目的地国家（地区）的**已确定日期、座位**的联程机（车、船）票。 |

#### 2.4 材料信息自动提取

完成材料校验后，系统通过多模态分析自动提取表单关键信息（如护照MRZ码信息、邀请函单位信息等），作为填表备选数据自动填充。

#### 2.5 人工审核触发机制

对于无法机审的特殊材料（如手写备注延期的护照等，具体判定标准待定，demo阶段需要后续讨论是否涉及），系统弹出提示："请联系现场工作人员进行人工确认"。（Demo阶段点继续即可）

---

### Step 3：补充信息收集（对话式填表）

**驱动方式**：大模型（自然语言理解 + 结构化信息提取）  
**交互形式**：在界面提供**可视进度条**，分模块引导。系统将未从材料中提取到的缺失信息，以单选、自然语言问答的方式向申请人提问。
（大模型依据表单需要的内容向用户提问，用户使用自然语言回答（打字），然后大模型根据用户的输入，填写对应的表单项，逐个模块进行问答）




#### 3.1 关键字段校验与比对规则（随填随审）

填表过程中，大模型将根据设定好的知识库实时执行硬性容错规则校验：

| 校验维度 | 对比数据源 | 容错与核验规则 |
|---------|-----------|--------------|
| **基本身份信息** | 护照信息页 | 表单填写的姓名、出生日期、性别、护照号必须与护照页（和MRZ码）**完全一致**。 |
| **出生地与国籍** | 护照信息页 | 出生地必须一致或符合隶属关系（如镇级）。非菲律宾出生需询问其他国籍；中国出生需询问曾有国籍。 |
| **签证参数约束** | 签证政策 | 签证有效期只能是 3, 6, 12 或 12的倍数；停留时间只能是 30 的倍数且 ≤ 180 天。 |
| **行程逻辑比对** | 机票 / 酒店订单 | 机票订单和酒店订单上**必须有申请人姓名**；表单填写的抵达/离境日期、航班号必须与机票一致；停留城市与酒店一致。 |
| **邀请人匹配** | 商务邀请函 | 签证种类为商务(M)或海员过境(G)的，邀请方必须是公司名称，且必须和邀请函上**一模一样**，必须为国内单位。 |
| **职业敏感词** | 填表录入数据 | 若填写为保姆/家政类（Nanny/Helper等），签证有效期只能为15天，需上传3张与雇主合照；若无业，需附解释说明。 |
| **家庭及联系方式** | 填表录入数据 | 本人电话中**必须包含至少一个菲律宾本地号码**（09开头的11位数字）；紧急联系人的电话和邮箱**不能与申请人相同**。 |
| **同行人选项** | 政策规定 | 偕行人（6.5项）必须填“否”。 |

> **处理机制**：一旦发现填报信息与规则冲突（如父亲年龄晚于申请人，或酒店城市与邀请函城市相隔极远），AI 会当时即指出错误要求申请人修改。

---

### Step 4：信息汇总确认

- 在问答流结束后，系统生成匹配于正式申请端的完整界面（表格）。
- 以可视化的表单形式向申请人展示所有已收集并校验通过的信息。
- 申请人最终核对并点击确认。

---

### Step 5：结构化数据输出

- 输出完整的 JSON 或 CSV 文件。
- 由签证中心员工查阅后进行授权。
- **后续落地**：调用自动化脚本，将包含上传材料在内的所有结构化数据，仿人工平移填写到外交部官方在线填表平台（visaforchina.cn）。
